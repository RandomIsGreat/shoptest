
Шаблон интернет магазина с возможностью регистрации новых пользователей, добавления товаров в заказ,
сохранения адреса. Основные функции присутствую на странице index.php.



Класс Database

Обеспечивает соединение с базой данных.
Через метод getInstance подключается к базе данных или возвращает текущее соединение.
Через метод query выполняет переданный запрос с переданным ассоциативным массивом значений.

Класс Connector

Обеспечивает соединение конкретного (имеющегося в бд) юзера с базой данных через емайл и пароль.
Метод runQuery выполняет проверку наличия юзера в в бд и сравнивает пароли, в случае несовпадения паролей
перенаправляет на страницу ввода пароля. В случае совпадения вызывает метод connectedRedirect.
Метод connectedRedirect генерирует куки для текущей сессии и текущего юзера. (в остальных страницах идет сравнение
параметра коннектед для куки и сессии, в случаи несовпадения сессия закрывается и происходит перенаправление).

Класс Factory

Должен был обеспечивать работу с разными объектами таблиц бд под общим интерфейсом (не придумао как правильно сделать).

Класс UserCreator наследован от Connector

Обеспечивает создание и сохранение данных нового юзера, прошедшего регистрацию путем переопределения родительского метода
runQuery

Класс AdressCreator наследован от Connector

Обеспечивает создание и связь адреса с текущим юзеро (переопределение метода runQuery)

Класс OrderActions наследован от Connector

(Сюда должен был быть переписан код из файла addtoorder.php в норамльный вид)

Cтраница index.php

Выводит зарегестрированному пользователю пять первых по возростанию цены товара с возможностью оставить комментарий
и добавить в корзину. Незарегестрированного пользователя перенаправляет на connector.php

Страница connector.php

Служит промежуточной страницей, проверяющей и производящей текущего пользователя.

Страница inputpassword.php

Служит для ввода входных данных пользователя, имеет ссылку на создание нового.

Страница new_account.php

Служит для создания нового пользователя, перенаправляет на creator.php

Страница creator.php

Обеспечивает создание нового пользователя

Страница addadress.php

Принимает параметры адреса пользователя и сохраняет его в базу со связью.

Страница addtoorder.php

Обеспечивает добавление товара в текущий заказ пользователя, через гет передается номер товара

Страница addcomment.php

Служит для добавления комментария к товару

Страница gooddescription.php

Непроверенная страница (глядя на нее решил переделать систему классов, наследованных от коннектора, посчитав их некорректной)
Должна выводить товар с описанием и комментариями ниже.
